# 如何开始整理你的 Markdown

如果你已经手里有很多 Markdown，最推荐的做法不是一次性重写，而是先完成整理和归类。

## 第一步：先按主题分类

你可以先粗分成这些目录：

- `guide/`：使用指南、入门说明
- `api/`：接口文档、字段说明
- `notes/`：个人笔记、研究记录
- `deploy/`：部署和运维

## 第二步：补标题和简介

优先给每篇文档补齐：

- `title`
- `description`

这会让导航与搜索结果更可读。

## 第三步：处理文档间链接

如果不同 Markdown 之间需要互相跳转，建议在正文中直接写站内链接：

```md
[返回系列 A 总览](/series-a/overview)
```

## 第四步：把首页当成可视化入口

首页不需要承载所有内容细节，更适合承担两件事：

- 展示站点气质
- 作为分类索引入口
